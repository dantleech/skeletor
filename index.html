Skeletor

Skeletor is a simple universal project skeleton generator

#FORK ME ON GITHUB

Introduction

Skeletor is a universal project skeletor generator.

  • Uses github repositories (example) as project skeletons.
  • The skeletons are defined in a skeletor.json file.
  • Uses simple {{ mustache }} templating.
  • Parameters are defined in the configuration and customized interactively.

Skeletor is for those people that create new projects on a regular basis and often find themselves wasting time copying / writing boilerplate code.

Installation

Install the phar and the public key (required for self updates)

$ curl -o skeletor.phar https://dantleech.github.io/skeletor/skeletor.phar
$ curl -o skeletor.phar.pubkey https://dantleech.github.io/skeletor/skeletor.phar.pubkey

Now make the PHAR executable and maybe you want to move it somewhere in your path:

$ chmod a+x skeletor
$ sudo mv skeletor /usr/local/bin/skeletor
$ sudo mv skeletor.pubkey /usr/local/bin/skeletor.pubkey

Generate a project

First you need to install a project template:

$ skeletor install dantleech/phplib.skel

Then you can generate a project:

$ skeletor generate

Creating your own repositories

The easiest way to start is probably just to fork an existing repository

The configuration

Your repository should have a skeletor.json file in its project root: 

{
    "title": "title for this skeleton",
    "description": "description for this skeleton",
    "params": {
        "param1": "default value 1",
        "param2": "default value 2"
    },
    "basedir": "skeletor",
    "files": {
        "lib": {
            "type": "dir"
        },
        "tests": { 
            "type": "dir"
        },
        "LICENSE": {}
        "lib/myfile.txt": {}
    }
}

Your directory structure should look like this:

/
  skeletor/
    lib/
      myfile.txt
    LICENSE
  skeletor.json

Note that in the configuration:

  • We defined lib and tests as directories - these will be explicitly created even if they are not in our repository.
  • We define a file map where the array key is the path of the file relative to basedir and the value is an object of options.
  • In the absence of a defined type the type is assumed to be template
  • We define parameters which can be substituted in files with the template type.

File types

Each file can have a type:

  • dir: A directory will be created, it does not have to exist in your repository.
  • file: The file located at the path will be copied verbatim.
  • template: The file will be processed and any {{ tokens }} will be replaced with the parameters corresponding to their values.

Templates

Files defined with a template type should look something like this:

Hello {{ author.name }}
This is an example template, that was created in {{ date.year }}.

Standard parameters

The following parameters are automatically available:

  • {{ date.year }}: The year.
  • {{ date.month }}: The month.
  • {{ date.day }}: The day.
